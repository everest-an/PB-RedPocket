Protocol Bank 项目需求文档 (PRD) v2.0
定位： 跨平台 Web3 支付/激励嵌入式组件
Slogan： 像发送消息一样发放 Web3 激励，无缝对账。

1. 核心产品愿景
打造一个无感的激励分发层。用户无需预先拥有钱包，在 Discord、Telegram、WhatsApp、GitHub 中即可通过社交身份直接领取奖励。资金统一托管，用户仅在提现或查看报表时才访问主站。

2. 关键创新：无感接入引擎 (Frictionless Engine)

社交身份即钱包 (Social Login & AA)：

o
通过 Privy / Magic Link / Dynamic 等技术，当用户在社交端点击“领取”时，系统基于其社交 ID 自动在后台生成一个 非托管型账户 (AA 钱包)。
o
o
用户无感： 无需记住私钥，无需下载 App，点击即领取。
o

跨平台连接器 (Cross-Platform Connectors)：

o
GitHub： 嵌入在 PR 评论区的互动卡片。
o
o
IM 应用 (TG/Discord/WhatsApp)： 通过 Bot 触发的极简消息卡片或 H5 链接。
o
o
WhatsApp 特色： 通过 Twilio 或 WhatsApp Business API，发送一个带有特定 Hash 的 H5 链接进行领取。
o

3. 功能模块详细设计
模块 A：嵌入式发放组件 (Distribution Component)

发放逻辑： 企业/管理员在主站或通过指令发起，设置：金额、个数、分类（如：营销费）、任务（如：Merge PR）。


分类锚定： 发放时强制要求打标签。

o
示例： /reward @dev1 50USDC --tag "BugFix"。
o

无钱包支持： 若接收方没有钱包，系统为其社交 ID 预留额度，待其点击链接时自动激活。

模块 B：企业管理后台 (Enterprise Dashboard)

明细看板： 实时流水的仪表盘，显示：发放时间、平台（TG/GH）、接收者 ID、状态（已领/待领）。


财务报表：

o
自动生成 P&L（盈亏平衡表）。
o
o
税务友好的凭证： 每一笔支出附带社交平台任务完成的截图或链接。
o

资金池管理： 企业充值稳定币到主账户，所有平台共用一个额度。

模块 C：用户取现中心 (User Portal)

登录： 用户回到 protocolbank.com，直接使用 TG/GitHub/WhatsApp 账号登录。


余额： 自动汇总在所有平台收到的红包/赏金。


提现： * Web3 路径： 提现到自己的 MetaMask/OKX 钱包。

o
法币路径： 通过 Off-ramp 合作伙伴（如 MoonPay/Transak）提取到银行卡（可选）。
o

4. 业务交互流程（以 WhatsApp 为例）
1.
发放： 项目方在后台生成一个 WhatsApp 奖励链接。
2.
3.
触达： 通过消息发送：“感谢参加活动，这是给您的 10 USDC 奖励：[link]”。
4.
5.
领取： 用户点击链接 -> 页面自动通过 OAuth 识别 WhatsApp 身份 -> 后台自动创建 AA 钱包 -> 资金到账。
6.
7.
沉淀： 企业后台自动记录一条：WhatsApp - [User Name] - 10 USDC - 营销支出。
8.

5. 技术架构方案
需求点	技术实现
底层钱包	Account Abstraction (ERC-4337)，支持 Gas 费代付，用户免 Gas 领取。
社交验证	Privy 或 Web3Auth，实现社交账号与链上地址的 1:1 映射。
多链分发	Base / Polygon / Sol (低 Gas 链，提升用户体验)。
消息分发	Webhook + Bot API (TG Bot, Discord Bot, GitHub Actions)。
数据同步	GraphQL (The Graph) 实时监控所有平台发出的智能合约事件。

6. 竞争优势 (The Edge)
1.
无需钱包（Wallet-less）： 极大降低了非 Crypto 用户的进入门槛。
2.
3.
全平台一致性： 开发者在 GitHub 领的钱，和他在 Discord 领的钱，登录主站发现是在同一个账上。
4.
5.
合规性（Compliance Ready）： 每一笔钱都有出处，解决了 Web3 公司最头疼的“支出证明”问题。
6.
